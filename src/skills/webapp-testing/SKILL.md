# Webapp Testing

Test web applications using Playwright with structured recognition-action patterns.

## When to Use

- Testing applications generated by agentflow
- Validating UI behavior after changes
- Automating end-to-end tests
- Regression testing web features

## The Recognition-Action Pattern

```
RECOGNIZE → DECIDE → ACT → VERIFY
```

1. **Recognize** — Identify UI elements and state
2. **Decide** — Determine what action to take
3. **Act** — Perform the action (click, type, navigate)
4. **Verify** — Confirm expected outcome

## Test Structure

```javascript
import { test, expect } from '@playwright/test';

test('user can complete checkout', async ({ page }) => {
  // Arrange
  await page.goto('/products');
  
  // Act
  await page.locator('[data-testid="product-1"]').click();
  await page.locator('[data-testid="add-to-cart"]').click();
  await page.locator('[data-testid="checkout"]').click();
  
  // Assert
  await expect(page.locator('[data-testid="success-message"]'))
    .toBeVisible();
});
```

## Selectors

Prefer these in order:
1. `data-testid` attributes — Most stable
2. Semantic roles — `getByRole('button', { name: 'Submit' })`
3. Text content — `getByText('Welcome')`
4. CSS selectors — Last resort, fragile

**Never use:**
- XPath with indices
- Auto-generated class names
- Positional selectors

## Test Data

- Use deterministic test data
- Reset state before each test
- Avoid dependencies between tests
- Mock external APIs when possible

## Integration with Agentflow

The `review` agent can automatically run webapp tests:

```javascript
// In review.js
if (cfg.testing?.webapp) {
  await runWebappTests(cfg);
}
```

Add to `agentflow.config.js`:

```javascript
testing: {
  webapp: true,
  playwright: {
    config: './playwright.config.js'
  }
}
```

## Guidelines

- One behavior per test
- Tests should be deterministic
- Fast execution (parallel when possible)
- Clear failure messages
- Screenshots on failure
